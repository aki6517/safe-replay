# テスト用メールテンプレート

## パターン1: 請求書送付依頼メール（優しい依頼）

**件名:**
```
【お願い】請求書の送付について
```

**本文:**
```
お世話になっております。
株式会社○○の△△です。

先日ご依頼いただいた作業について、請求書の送付をお願いいたします。
請求書が届いていないため、お支払いの手続きができずに困っております。

お忙しいところ恐縮ですが、できるだけ早くご送付いただけますでしょうか。
よろしくお願いいたします。

株式会社○○
△△
```

## パターン2: 納期遅延に対する怒りのメール

**件名:**
```
【至急】納期遅延について - 非常に困っています
```

**本文:**
```
お世話になっております。
株式会社○○の△△です。

先日、納期を3月15日とお約束いただいた件について、本日3月20日になってもまだ納品物が届いておりません。
お客様への納期も迫っており、非常に困っています。

納期を守っていただくことは、ビジネスの基本だと思います。
このような状況が続くと、今後の取引についても見直さざるを得ません。

至急、状況をご連絡いただき、具体的な納期をご提示ください。
本日中にご返信をお願いいたします。

株式会社○○
△△
```

## パターン3: 品質不良に対するクレームメール

**件名:**
```
【重要】納品物の品質について - 再発防止をお願いします
```

**本文:**
```
お世話になっております。
株式会社○○の△△です。

先日納品いただいた商品について、品質に問題があることが判明いたしました。
具体的には、仕様書と異なる仕様で納品されており、お客様からも苦情をいただいております。

このような品質不良が続くと、お客様からの信頼を失うことになりかねません。
再発防止のための対策を講じていただき、今後このようなことがないようお願いいたします。

至急、ご対応方法をご連絡いただきたく存じます。
よろしくお願いいたします。

株式会社○○
△△
```

```
至急、状況をご確認いただき、復旧の見込みをお知らせください。

本日中にご返信をお願いいたします。

株式会社○○
△△
```

## 使用方法

1. 上記のメール内容をコピー
2. 自分のメールアドレス宛に送信
3. `npm run test-gmail-polling` を実行
4. LINE通知で語りかけ形式のメッセージが表示されることを確認

## テストのポイント

- **パターン1**: 優しい依頼メール → LINEBOTが「困っているみたいだよ」と優しく伝えるか確認
- **パターン2**: 怒りのメール → LINEBOTが「怒っているわけじゃなくて、困っているみたいだよ」と伝えるか確認
- **パターン3**: クレームメール → LINEBOTが「責められているわけではない」と伝えるか確認
- **パターン4**: 緊急メール → LINEBOTが緊急性を和らげて伝えるか確認
- **パターン5**: 確認依頼メール → LINEBOTが優しく提案するか確認

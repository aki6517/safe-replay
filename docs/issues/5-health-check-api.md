### 背景 / 目的

システムの生存確認と詳細ヘルスチェックエンドポイントを実装し、監視・運用の基盤を整える。

- 依存: #4
- ラベル: `backend`, `monitoring`

### スコープ / 作業項目

- シンプルなヘルスチェックエンドポイントの実装
- 詳細ヘルスチェックエンドポイントの実装（DB・Redis接続確認）
- Service Key認証の実装
- 各コンポーネントの状態確認ロジック

### ゴール / 完了条件（Acceptance Criteria）

- [ ] `GET /api/v1/health`を実装（シンプルな生存確認）
- [ ] `GET /api/v1/health/deep`を実装（DB・Redis接続確認）
- [ ] Service Key認証を実装
- [ ] 各コンポーネントの状態を返す
- [ ] エラー時に適切なHTTPステータスを返す

### テスト観点

- リクエスト: `GET /api/v1/health`と`GET /api/v1/health/deep`にアクセス
- 検証方法: 正常時は200、異常時は適切なHTTPステータスが返ることを確認
